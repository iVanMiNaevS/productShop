# Как развернуть и запустить

## next
1) npm i
2)  заполнить .env для работы с jwt токенами и и защищёнными запросами strapi смотреть env.example.
Чтобы указать ADMIN_TOKEN нужно сгенерировать токен в strapi с доступом для update пользователя.
3) npm run dev

## strapi
1) npm i develop
2) создать категории и продукты. Привязать продукты к категориям
# Стэк

## frontend
next (ts, sass)

## backend
1) strapi (cms) для хранения данных и манипуляций над ними

2) api next для запросов посредников между front и strapi. Нужно для безопасности и проверки кто делает запросы.

# Авторизация и безопасность

Сделана на cookie и jwt

Защищённые запросы делаются на api next, там проверяется token из кук (токен пользователя) и делается запрос к strapi 

Запрос к Strapi делается с переданным токеном из env next ADMIN_TOKEN тем самым позволяя обращаться к защищённым маршрутам Strapi
только серверу next

## Схема

клиент -> next api (проверяет token пользователя) -> запрос в strapi (с токеном ADMIN_TOKEN)
